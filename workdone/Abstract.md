HOLOID is a full-stack Hospital Bed and Resource Management System (HBRMS) designed to improve emergency healthcare coordination during pandemics, natural disasters, and regional demand surges. Existing hospital workflows often rely on fragmented communication, delayed updates, and manual data reconciliation, which can slow patient transfers and reduce decision accuracy. HOLOID addresses these gaps through a centralized digital platform that provides secure, real-time visibility of hospital capacity and resource utilization.

The system integrates a React-based frontend with a Node.js/Express backend and MongoDB data layer. Core modules include role-based authentication with email activation, hospital resource inventory management, transfer request orchestration, command-center analytics, and notification services. Bed availability is tracked at ward level, while logistics endpoints support discovery of suitable hospitals and transfer status tracking. Real-time event broadcasting enables faster operational awareness for connected stakeholders. Audit-oriented data handling and standardized API responses support traceability and consistency across workflows.

By digitizing key coordination processes, HOLOID reduces manual errors, shortens response time, and supports data-driven decisions for administrators, doctors, and government officials. The platform demonstrates how a modular full-stack architecture can strengthen healthcare resilience by unifying monitoring, communication, and resource allocation in a single operational environment. Overall, the project offers a practical and scalable approach to hospital resource governance under high-pressure conditions.
